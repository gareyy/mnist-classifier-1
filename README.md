# MNIST-classifier-1

basic mnist classifier with tkinter based frontend

Classifier model based on a neural network, with 2 hidden layers, each with a size of 512 parameters, each using ReLU. Trained using stochastic gradient descent with a really tiny but fixed learning rate. Stopping point for training happens when test dataset accuracy is greater than 95% or when number of training epochs is more than 100.

The number drawing frontend is very basic, and uses a single brush that does not have any blur on the edges of the brush. This means that drawings that come from the frontend are turned into matrices that only have values 0 and 1, and never anything inbetween.

Because of my frontend issues, I had to modify the MNIST dataset. `custom_torch_train.py` contains a custom torch Dataset class that clamps the values in the image data to either be 0 or 1, which when viewed, are more close to datapoints generated by a user drawing in the frontend.
